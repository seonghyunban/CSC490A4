\subsubsection{Reward A: Format Compliance}
\nestindent

\textbf{What it is.}
A binary reward that returns 1.0 if the response contains a parseable answer in the expected \texttt{\#\#\#\# <number>} format, and 0.0 otherwise:
\[
R_A(\text{response}) =
\begin{cases}
1.0 & \text{if } \texttt{\#\#\#\# (-?[0-9.,]+)} \text{ matches} \\
0.0 & \text{otherwise}
\end{cases}
\]
A score of 1.0 means the response contains an extractable numeric answer, regardless of whether it is correct.

\textbf{Why we chose it.}
To address the 60.2\% extraction failure rate identified in Section~1.2: responses without a parseable answer receive zero correctness reward, blocking all gradient signal.
Format compliance decouples format learning from reasoning, providing non-zero advantage variance even when all 16 samples are incorrect.

Format compliance rewards are standard in RLVR implementations for GSM8K~\cite{gsm8k_rlvr, verl, unsloth}.
We considered a calculator usage reward (rejected---risks incentivizing meaningless tool calls) and a reasoning structure reward (rejected---models learn to repeat correct but unnecessary reasoning steps to inflate structure scores~\cite{gao2024reward}).
