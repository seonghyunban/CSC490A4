Part 3 trained a base model on GSM8K using a single binary correctness reward via REINFORCE.
While this establishes a functional RL pipeline, a single reward provides limited learning signal---responses are either fully correct or receive zero reward, leaving no gradient for partial progress.
Part 4 addresses this by designing five additional reward functions (A--E) that target specific failure modes observed in the trained model.
All additional rewards supplement the binary correctness reward; they never replace it.
Rewards A and B are pre-designed from literature and baseline error analysis: format compliance (addressing the \textasciitilde11\% of responses that lack a parseable answer format) and numeric proximity (providing graded signal for the \textasciitilde74\% that format correctly but produce wrong answers).
Rewards C, D, and E are designed after exploratory data analysis of the baseline RL outputs, targeting failure patterns identified therein.
We evaluate these rewards through a ten-run ablation: a Baseline (correctness only), five single-reward runs (correctness + one additional reward each), three combination runs, and a Comb.\ All run (correctness + all five).
Each run uses the same SFT checkpoint and identical RL configuration, varying only the reward composition.
We compare runs on Pass@1, Pass@8, extraction failure rate, and a rule-based mistake taxonomy, and report training dynamics from W\&B logs.
